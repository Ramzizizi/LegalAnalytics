# Архитектура проекта

Конструктор типовых юридических документов — веб-приложение на Django, которое
позволяет юристу выбрать шаблон, заполнить форму и получить готовый `.docx`/`.pdf`.

## Обзор

```
                  ┌─────────────────────────────────────────────┐
                  │                  Браузер                     │
                  │      Bootstrap 5 + HTMX (без React/Vue)      │
                  └───────────────────┬─────────────────────────┘
                                      │ HTTP / HTMX
                  ┌───────────────────▼─────────────────────────┐
                  │              Django 5.1 (gunicorn)           │
                  │                                              │
                  │  apps/accounts    — роли и доступ            │
                  │  apps/catalog     — контрагенты, юристы      │
                  │  apps/templates_engine — шаблоны + движок    │
                  │  apps/documents   — форма → генерация → журнал│
                  └─────────┬───────────────────────┬───────────┘
                            │                       │
              ┌─────────────▼──────┐   ┌────────────▼─────────────┐
              │   PostgreSQL       │   │   LibreOffice headless    │
              │  (SQLite локально) │   │   .docx → .pdf (subprocess)│
              └────────────────────┘   └──────────────────────────┘
```

Статику (`CSS/JS`) в продакшне раздаёт **WhiteNoise** прямо из gunicorn — без nginx.

## Приложения (Django apps)

| App | Ответственность | Ключевые файлы |
|-----|-----------------|----------------|
| `accounts` | Роли (Django Groups), декораторы доступа | `utils.py`, `management/commands/setup_roles.py` |
| `catalog` | Справочник контрагентов и юристов, валидатор ИНН | `models.py`, `validators.py` |
| `templates_engine` | Word-шаблоны, переменные, движок генерации | `models.py`, `services.py` |
| `documents` | Динамическая форма, генерация, журнал, скачивание | `views.py`, `forms.py`, `models.py` |

## Модель данных

```
┌──────────────────┐         ┌──────────────────────┐
│   Контрагент     │         │   ШаблонДокумента     │
│──────────────────│         │──────────────────────│
│ наименование     │         │ название             │
│ тип (юр/физ)     │         │ тип                  │
│ инн  ◄── валидатор│        │ файл (.docx)         │
│ кпп, адрес       │         │ активен              │
│ телефон, email   │         └──────────┬───────────┘
└────────┬─────────┘                    │ 1
         │                              │
         │                              │ N
         │                   ┌──────────▼───────────┐
         │                   │     Переменная       │
         │                   │──────────────────────│
         │                   │ ключ (тег в шаблоне) │
         │                   │ подпись (в форме)    │
         │                   │ тип_данных           │
         │                   │ обязательное         │
         │                   │ значение_по_умолчанию│
         │                   │ порядок              │
         │                   └──────────────────────┘
         │
         │ N                      N │           N │
┌────────▼────────────────────────▼─────────────▼────────┐
│              СгенерированныйДокумент                     │
│─────────────────────────────────────────────────────────│
│ шаблон (FK, PROTECT)                                     │
│ автор (FK User, PROTECT)                                 │
│ контрагент (FK, PROTECT)                                 │
│ значения (JSON — снимок данных формы)                    │
│ файл_docx, файл_pdf                                      │
│ статус (черновик / готов / ошибка)                       │
│ ошибка (текст исключения, если статус='ошибка')          │
│ создан, изменён                                          │
└─────────────────────────────────────────────────────────┘
```

**Почему так:**
- `Переменная` привязана к `ШаблонДокумента` — это позволяет строить форму
  динамически под каждый шаблон без правки кода (`ДинамическаяФорма`).
- `СгенерированныйДокумент.значения` хранит JSON-снимок введённых данных —
  документ можно воспроизвести, даже если контрагент потом изменится.
- FK с `on_delete=PROTECT` — нельзя удалить шаблон/контрагента, по которому уже
  сгенерированы документы (защита истории).

## Конвейер генерации

Точка входа — `apps/documents/views.py::_генерировать`.

```
1. Форма валидна (ДинамическаяФорма.is_valid)
        │
2. форма.получить_контекст()
        │   • поля формы + авто-поля контрагента (contragent_name, _inn, _kpp, _address)
        │   • даты → строки ДД.ММ.ГГГГ, None → ''
        ▼
3. transaction.atomic:
        │
        ├─ создаётся запись СгенерированныйДокумент (статус='черновик')
        │
        ├─ render_docx(шаблон.файл.path, контекст, docx_path)    ← docxtpl + Jinja2
        │       сохраняет .docx в media/documents/docx/
        │
        ├─ если LibreOffice доступен:
        │       convert_to_pdf(docx_path)                         ← subprocess
        │       сохраняет .pdf в media/documents/pdf/
        │
        ├─ статус='готов'
        │   (любое исключение → статус='ошибка', текст в поле .ошибка)
        │
        └─ doc.save()
```

Имена файлов содержат `uuid4`-суффикс, чтобы избежать коллизий.

**`render_docx`** (`templates_engine/services.py`) — обёртка над `docxtpl.DocxTemplate`:
подставляет значения в Word-шаблон по Jinja2-тегам `{{ ключ }}`.

**`convert_to_pdf`** — запускает `libreoffice --headless --convert-to pdf` через
`subprocess`. Каждый запуск использует свой профиль (`-env:UserInstallation`),
чтобы параллельные воркеры gunicorn не конфликтовали за общий lock-файл.

## Роли и доступ

Реализованы через Django Groups (`accounts/management/commands/setup_roles.py`).

| Роль | Шаблоны | Создание документов | Просмотр |
|------|---------|---------------------|----------|
| **Администратор шаблонов** | управляет | да | свои |
| **Юрист** | — | да | свои |
| **Помощник** | — | да | свои |

Декоратор `требует_группу(*роли)` из `accounts/utils.py` закрывает вьюхи.
Суперпользователь проходит любую проверку. Каждый пользователь видит и скачивает
**только свои** документы (`автор=request.user` во всех queryset).

## Безопасность

- `SECRET_KEY` обязателен при `DEBUG=False` (иначе приложение не стартует).
- `media/` раздаётся напрямую только в `DEBUG`; в проде скачивание идёт через
  вьюху `download` с проверкой автора и защитой от path traversal.
- Скачивание валидирует, что путь файла лежит внутри `MEDIA_ROOT`.
- Контейнер работает от непривилегированного пользователя `appuser`.
- Секреты только через `.env` (не коммитятся).
