# Тестирование

## Автотесты

```bash
pytest                       # все тесты
pytest -v                    # подробный вывод
pytest tests/test_validators.py   # один файл
pytest -k инн                # тесты по подстроке
```

**Покрытие:** 87 тестов, разбивка по модулям:

| Файл | Тестов | Что проверяет |
|------|--------|---------------|
| `test_validators.py` | 27 | ИНН (валидные/невалидные, формат), суммы |
| `test_access.py` | 14 | роли, доступ к вьюхам, изоляция документов |
| `test_services.py` | 12 | `render_docx`, `convert_to_pdf` (вкл. ошибки) |
| `test_models.py` | 11 | создание моделей, `__str__`, уникальность |
| `test_journal.py` | 11 | пагинация, фильтры, поиск, HTMX, изоляция |
| `test_documents.py` | 7 | генерация, скачивание (вкл. чужой → 404), форма |
| `test_templates.py` | 5 | все .docx-шаблоны рендерятся без незакрытых тегов |

> 3 теста `convert_to_pdf` пропускаются (`skip`), если LibreOffice не установлен
> локально. В Docker они проходят. Ошибочные пути конвертации тестируются через
> мок (`test_services.py::TestConvertToPdfMocked`) и работают без LibreOffice.
