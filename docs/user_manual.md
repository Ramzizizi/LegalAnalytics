## 1. Вход и выход из системы

### Вход

1. Откройте браузер и перейдите по адресу `http://localhost:8000`.
2. Система автоматически перенаправит вас на страницу входа.
3. Введите логин и пароль, нажмите **«Войти»**.

![Форма входа](screenshots/login.png)

После входа в правом верхнем углу отображается ваше имя и роль в системе.

**Демо-пользователи для проверки:**

| Логин | Пароль | Роль |
|---|---|---|
| `editor` | `editor123` | Редактор базы |
| `analyst` | `analyst123` | Аналитик |
| `reader` | `reader123` | Читатель |

### Выход

Нажмите на своё имя в правом верхнем углу → выберите **«Выйти»**.

## 2. Навигация

После входа в верхней панели доступны разделы в зависимости от вашей роли:

```
[Правовая аналитика]  Поиск  База ▼  Конструктор  Аналитика     [Имя пользователя ▼]
                                │
                         ├── Нормы права
                         ├── Судебная практика
                         └── Заключения
```

- **Поиск** — полнотекстовый поиск по всей базе.
- **База** — прямой переход к спискам норм, практики или заключений.
- **Конструктор** — сборка правового заключения (только Аналитик и Редактор).
- **Аналитика** — дашборд с диаграммами (только Аналитик и Редактор).

## 3. Поиск по базе

Поиск — основной инструмент работы с системой. Он охватывает сразу все типы материалов: нормы права, судебную практику и заключения.

### 3.1. Простой поиск

1. Нажмите **«Поиск»** в навигации или перейдите на `/search/`.
2. Введите запрос в строку поиска.
3. Нажмите **«Найти»** или клавишу `Enter`.

Результаты появляются без перезагрузки страницы. Каждый результат содержит:
- цветной бейдж типа материала (синий — НПА, зелёный — СУД, жёлтый — ЗАК);
- заголовок и подзаголовок;
- фрагмент текста с **выделенными совпадениями**;
- теги документа.

### 3.2. Особенности поиска

Система понимает **морфологию русского языка**: запрос «договоры» найдёт документы со словом «договор», «договора», «договором» и т.д.

**Специальные возможности поисковой строки:**

| Что ввести | Что найдёт |
|---|---|
| `неустойка` | Все документы, содержащие слово «неустойка» и его словоформы |
| `договор аренды` | Документы, где встречаются оба слова |
| `"надлежащим образом"` | Точную фразу «надлежащим образом» |
| `-алименты` | Исключает документы со словом «алименты» |
| `алименты OR неустойка` | Документы с любым из слов |

### 3.3. Сброс поиска

Нажмите кнопку **×** справа от строки поиска, чтобы очистить запрос.

## 4. Фасетная фильтрация

После выполнения поиска в левой боковой панели появляются фильтры для уточнения результатов.

### 4.1. Фильтр по типу материала

Нажмите на один из вариантов:
- **Все типы** — без фильтрации (по умолчанию)
- **Нормы права** — только НПА
- **Практика** — только судебные решения
- **Заключения** — только правовые заключения

### 4.2. Фильтр по отрасли права

В панели «Отрасль права» отображаются только те отрасли, по которым есть результаты. Рядом с каждой — количество документов.

Нажмите на отрасль, чтобы отфильтровать. Повторный клик — снимает фильтр.

### 4.3. Фильтр по тегам

Панель «Теги» показывает топ-15 тегов из текущих результатов поиска. Нажмите на тег — результаты отфильтруются.

### 4.4. Фильтр по дате

1. Заполните поля **«С»** и/или **«По»** в панели «Дата».
2. Нажмите **«Применить»**.
3. Для сброса нажмите **«Сбросить дату»**.

### 4.5. Комбинирование фильтров

Фильтры суммируются: можно одновременно применить тип + отрасль + тег + дату. Активные фильтры отображаются в виде бейджей над результатами. Кнопка **×** на бейдже снимает конкретный фильтр.

## 5. Просмотр карточек

### 5.1. Переход к карточке

- Из результатов поиска — нажмите на любой результат.
- Из списков (Нормы / Практика / Заключения) — нажмите на запись.

### 5.2. Карточка нормы права

На карточке нормы отображается:
- вид НПА (Федеральный закон, Кодекс и т.д.);
- отрасль права;
- номер статьи и полный текст;
- дата вступления в силу и источник;
- теги (каждый тег — ссылка на поиск по нему);
- кнопка **«Найти похожие»** — запускает поиск по номеру статьи;
- кнопка **«В конструктор»** — добавляет норму в корзину заключения *(только Аналитик и Редактор)*;
- блок **«Судебная практика»** — связанные судебные решения;
- блок **«Правовые заключения»** — связанные заключения;
- **граф связей**.

### 5.3. Карточка судебного дела

На карточке практики отображается:
- суд, номер дела, дата решения;
- тезис и полный текст;
- кнопка **«В конструктор»**;
- связанные нормы права;
- связанные заключения;
- граф связей.

### 5.4. Карточка правового заключения

На карточке заключения отображается:
- автор и дата создания;
- теги;
- полный текст заключения;
- связанные нормы и практика;
- граф связей.

> **Примечание для Читателей:** заключения с отметкой «Общедоступное» видны всем пользователям. Остальные заключения доступны только Аналитикам и Редакторам.

### 5.5. Списки материалов

Разделы **Нормы права**, **Судебная практика** и **Заключения** содержат полные списки с постраничной навигацией (по 20 записей). Для норм и практики доступна фильтрация по отрасли.

## 6. Граф связей

На каждой карточке внизу отображается интерактивный граф связей.

**Обозначения узлов:**

| Цвет / форма | Тип |
|---|---|
| Синий (крупный) | Текущий документ (центр графа) |
| Синий (обычный) | Связанная норма права |
| Зелёный | Связанное судебное дело |
| Жёлтый | Связанное заключение |

**Управление графом:**

| Действие | Результат |
|---|---|
| Клик по узлу | Переход на карточку документа |
| Наведение на узел | Всплывающая подсказка с названием |
| Перетаскивание | Перемещение узла |
| Колёсико мыши | Масштабирование |
| Перетаскивание фона | Прокрутка графа |

## 7. Конструктор заключения

> Доступно только **Аналитикам** и **Редакторам**.

Конструктор позволяет собрать правовое заключение из материалов базы и экспортировать его в формате .docx.

### 7.1. Добавление материалов в корзину

**Способ 1 — из карточки документа:**
1. Откройте карточку нормы или судебного дела.
2. Нажмите кнопку **«В конструктор»**.
3. Документ добавляется в корзину. Счётчик в навигации обновляется.

**Способ 2 — из результатов поиска:**
1. Найдите нужный материал через поиск.
2. Откройте его карточку и нажмите **«В конструктор»**.

> Один и тот же документ можно добавить только один раз — дублей не будет.

### 7.2. Работа с корзиной

Перейдите в **Конструктор** (кнопка в навигации).

На странице конструктора отображаются:
- список добавленных **норм** с названием и отраслью;
- список добавленного **судебной практики** с номером дела и судом;
- кнопки **«Удалить»** у каждого элемента (корзина обновляется без перезагрузки);
- кнопка **«Очистить корзину»**.

### 7.3. Составление и экспорт заключения

1. На странице конструктора заполните форму:
   - **Тема заключения** — заголовок итогового документа.
   - **Правовой вопрос** — постановка вопроса (необязательно).
   - **Текст заключения** — ваш анализ и вывод.

2. Нажмите **«Скачать .docx»**.

3. Браузер загрузит файл Word с именем, совпадающим с темой заключения.

**Структура сгенерированного документа:**

```
[Тема заключения]
Дата составления: ДД.ММ.ГГГГ
Составитель: [Имя пользователя]

Правовой вопрос
  [Текст вопроса]

Применимые нормы права
  • [Статья] (Вид НПА)
    [Выдержка из текста нормы]

Судебная практика
  • [Номер дела] — [Суд], [Дата]
    [Тезис решения]

Заключение
  [Текст заключения]
```

> **Совет:** корзина сохраняется в течение сессии. После выхода из системы корзина очищается. Если нужно сохранить промежуточную работу — экспортируйте документ.

## 8. Дашборд аналитики

> Доступно только **Аналитикам** и **Редакторам**.

Перейдите в раздел **«Аналитика»** в навигации.

### 8.1. KPI-счётчики

В верхней части страницы отображаются четыре счётчика текущего состояния базы:
- **Нормы** — общее количество НПА в базе
- **Практика** — количество судебных решений
- **Заключения** — количество правовых заключений
- **Отрасли** — количество отраслей права

### 8.2. Диаграммы

| Диаграмма | Описание |
|---|---|
| **Нормы по отраслям** (столбчатая) | Распределение НПА по отраслям права; топ-10 |
| **Нормы по видам** (круговая) | Доли законов, кодексов, указов, постановлений и т.д. |
| **Практика по отраслям** (горизонтальная) | Количество дел в каждой отрасли |
| **Динамика практики** (линейная) | Количество судебных решений по годам |

Наведите курсор на сегмент диаграммы — появится всплывающая подсказка со значением.

## 9. Администрирование базы

> Доступно только **Редакторам** и суперпользователям.

### 9.1. Доступ к панели администрирования

Нажмите на своё имя в правом верхнем углу → **«Администрирование»**, или перейдите напрямую на `/admin/`.

### 9.2. Добавление нормы права

1. В admin-панели выберите **Knowledge → Нормы → + Добавить**.
2. Заполните поля:
   - **Название** — полное название НПА.
   - **Вид НПА** — выберите из списка (Федеральный закон, Кодекс и т.д.).
   - **Статья** — номер статьи (например, «ст. 309 ГК РФ»).
   - **Текст нормы** — полный текст статьи.
   - **Источник** — ссылка на КонсультантПлюс или официальное издание.
   - **Дата вступления в силу** — в формате ДД.ММ.ГГГГ.
   - **Отрасль права** — выберите из справочника.
   - **Теги** — введите через запятую (создаются автоматически).
3. Нажмите **«Сохранить»**.

### 9.3. Добавление судебного дела

1. **Knowledge → Судебная практика → + Добавить**.
2. Заполните:
   - **Суд** — полное наименование суда.
   - **Номер дела** — в формате «А53-12345/2022».
   - **Дата решения**.
   - **Тезис** — краткое содержание решения (до 500 символов).
   - **Текст** — полный текст или ссылка.
   - **Отрасль права**.
   - **Связанные нормы** — выберите нормы, которые суд применил (блок справа).
   - **Теги**.
3. Нажмите **«Сохранить»**.

### 9.4. Добавление правового заключения

1. **Knowledge → Правовые заключения → + Добавить**.
2. Заполните:
   - **Тема заключения**.
   - **Текст заключения**.
   - **Автор** — пользователь системы.
   - **Общедоступное** — установите флаг, если заключение должны видеть Читатели.
   - **Связанные нормы** и **связанная практика**.
   - **Теги**.
3. Нажмите **«Сохранить»**.

### 9.5. Управление отраслями права

**Catalog → Отрасли права → + Добавить**.

Поля: название (уникальное), slug (автозаполняется), описание.

### 9.6. Управление пользователями

**Authentication → Пользователи** — создание и редактирование учётных записей.

Для назначения роли:
1. Откройте пользователя.
2. В блоке **«Группы»** выберите нужную роль (Редактор базы / Аналитик / Читатель).
3. Нажмите **«Сохранить»**.

## 10. Роли и права доступа

| Функция | Читатель | Аналитик | Редактор |
|---|:---:|:---:|:---:|
| Вход в систему | ✓ | ✓ | ✓ |
| Поиск и просмотр | ✓ | ✓ | ✓ |
| Публичные заключения | ✓ | ✓ | ✓ |
| Непубличные заключения | — | ✓ | ✓ |
| Конструктор заключения | — | ✓ | ✓ |
| Дашборд аналитики | — | ✓ | ✓ |
| Панель администрирования | — | — | ✓ |
| Добавление / редактирование записей | — | — | ✓ |

Если вы попытаетесь открыть страницу, недоступную для вашей роли, система отобразит сообщение об ошибке «Доступ запрещён» (код 403).

## Типичные вопросы

**В. Поиск не даёт результатов по очевидному запросу.**  
О. Убедитесь, что база наполнена данными. Попробуйте другую словоформу или сократите запрос до одного ключевого слова.

**В. Кнопки «В конструктор» и «Аналитика» не отображаются.**  
О. Эти функции доступны только ролям «Аналитик» и «Редактор базы». Обратитесь к администратору для назначения нужной роли.

**В. Скачанный .docx не открывается.**  
О. Убедитесь, что на вашем компьютере установлен Microsoft Word или LibreOffice.

**В. Граф связей пустой.**  
О. Граф строится по M2M-связям, которые задаются в admin-панели при добавлении практики и заключений. Если связи не указаны — граф не отобразит рёбра.

**В. Фасеты (счётчики) не появляются.**  
О. Фасеты отображаются только при наличии поискового запроса. Введите текст в строку поиска.
