# Проект: Конструктор типовых юридических документов

## Назначение
Веб-приложение: юрист выбирает шаблон → заполняет форму →
система подставляет данные → выгружает готовый .docx и .pdf.
Учебный проект (Цифровая кафедра ДГТУ). Право РФ: ГК, ГПК, АПК.

## Стек
- Django 5.1, Python 3.12 (в Docker) / 3.14 (локально)
- docxtpl + Jinja2 — генерация .docx из Word-шаблонов
- LibreOffice headless — конвертация .docx → .pdf
- PostgreSQL (prod/docker), SQLite (локально)
- Bootstrap 5 + HTMX, django-htmx
- Docker Compose

## Структура приложений
- `apps/catalog/` — Контрагент, Юрист
- `apps/templates_engine/` — ШаблонДокумента, Переменная; services.py с render_docx и convert_to_pdf
- `apps/documents/` — СгенерированныйДокумент, формы, HTMX
- `apps/accounts/` — роли, права, аутентификация

## Соглашения
- Код и комментарии — по-русски там, где это доменные термины.
- Все строки/документы — UTF-8.
- Валидаторы реквизитов (ИНН, даты, суммы) — в `apps/catalog/validators.py`.
- Тесты обязательны для движка генерации и валидаторов (pytest).
- Никаких секретов в коде — только через .env.
- Шаблоны Word (.docx) хранятся в `word_templates/`, путь берётся из БД (поле `ШаблонДокумента.файл`).

## Команды
- Запуск локально: `python manage.py runserver` (SQLite)
- Запуск в Docker: `docker compose up`
- Тесты: `pytest`
- Миграции: `python manage.py makemigrations && python manage.py migrate`
